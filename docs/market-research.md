## 一、商业产品怎么做的

### 主流五层架构

```
应用层    商机看板 智能搜索 订阅推送 AI助手 竞品分析
生成层    标书生成 客户摘要 切入建议 周报自动化
AI分析层  NLP语义 线索评分 意图刻画 商机匹配 策略优化
数据采集层 RPA自动采集 日均10万+条  多源:招标/新闻/政策/社媒/招聘
基建层    清洗去重 实体识别 知识图谱 主数据标准化
```

### 核心能力（这些是标配）

| 能力 | 做法 |
|------|------|
| 数据规模 | 400万+存量，日均7000条增量，覆盖全国95%数据源 |
| 全文搜索 | Elasticsearch 分面聚合 + 全文检索，非 SQLite FTS5 |
| 企业画像 | 六维（资质/人员/业绩/报价/关系/风险），知识图谱 |
| 商机预测 | ARIMA/GRU-LSTM 时序模型，提前3-6月预测潜在项目 |
| 竞品追踪 | 对手中标规律、报价区间、市场集中度分析 |
| 智能标书 | LLM+RAG企业知识库，自动匹配招标要求 |
| 推送通知 | 飞书/企微/钉钉/微信/邮件多通道，秒级触发 |
| 可视化 | GIS热力图、雷达图、竞争力仪表盘、ECharts |

### 成熟产品

- **建鱼标讯** — 首月150条商机，决策周期缩短40%
- **建设库** — 400万+数据池，复杂查询P95<1s
- **世界工程网** — "项目-企业-人"三级穿透，提前4月卡位
- **深圳大数据平台** — 7种分析算法（ARIMA/K-Means/PCA等）

## 二、GitHub 开源项目现状

### 现有项目

| 项目 | 做什么 | 缺什么 |
|------|--------|--------|
| CrawlerForCCGP | 爬ccgp.gov.cn + Excel | 无搜索 |
| CrawlerBidding | 同上 + 邮件通知 | 无分析 |
| Web_Scraping | 抓中标公示 → CSV | 极简 |
| Spider(多源) | 药监/招标/期刊多源 | 无索引 |

### GitHub 上所有人的共同特点

1. **只能爬** — 抓数据 → Excel/CSV → 结束。没有分析闭环
2. **单源** — 每个项目只爬一个网站
3. **无搜索** — 全文搜索一个都没有，靠网页自带搜索接口
4. **无标签** — 不打优先级/阶段/评分
5. **无投标书** — 完全没有标书生成这个能力

**换句话说：你的 bid_tool 已经比 GitHub 上所有公开项目都完整。**

## 四、诚实结论

### 不值得做的

- **Elasticsearch** — 个人工具 SQLite FTS5 够用，ES 太重
- **知识图谱** — 需要海量企业数据做底，个人没法积累
- **ML预测模型** — 没数据量支撑，做出来也是玩具
- **GIS热力图** — 数据就几个省，画地图没意义

### 可以做的（按性价比排序）

| # | 方向 | 原因 | 跟市场差异点 |
|---|------|------|------------|
| 1 | 多源爬虫 | ccgp+ggzy+cebpubservice，数据量翻几倍 | 市面工具都是单源 |
| 2 | 数据分析报告 | 地区热度/类型趋势/预算分布，用已有数据做 | 拿 `report.py` 升级 |
| 3 | 企业竞品对比 | 每次中标记录里提取中标方，积累竞品库 | 市面六维画像的简约版 |
| 4 | 通知推送 | 新高优商机 → 企业微信/钉钉 Webhook | 订阅监控的核心 |
